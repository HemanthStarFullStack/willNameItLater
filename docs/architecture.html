System Architecture · Prototype

A private assistant that lives on your device

One chat box. A small language model, a searchable personal memory, and a rule for every question: take the cheapest of three paths and never make up your facts.

On-device · nothing leaves except the web path
The brain — two models, split by job

🗣️ Generator small LLM · fits in memory

Does all language work — answering, classifying, summarizing. Small and quantized so it sits entirely in the device's memory; "thinking" decoding off for speed.

🧭 Embedder vectors · always on

Turns text into a meaning fingerprint. Cheap. Powers both memory search and the routing decision — so the big model isn't spent on similarity math.

Personal memory — and how facts get in

📓 The notebook stored on device · survives restarts

Every fact is kept as { text · label · meaning-fingerprint }. Fingerprinted once when saved, so answering later is fast.

Why it's built this way

Least work per question

Always one fingerprint; one answer for chat; a second model call only when honesty-checking or web-summarizing actually needs it.

Don't trust a small model to judge itself

Deciding "is this about me / do I need the web / did I stay honest" is handed to fixed rules and a similarity threshold, not the model — which is unreliable at those.

Right-size the brain to the device

The limit is memory, not cleverness. A model too big to fully fit spills to the slow processor (seconds → minutes). Rule: largest model that fully fits, chosen per device.

Private by construction

Memory and chat never leave the device. The only outbound traffic is a web search you triggered by asking something live.

Prototype vs. the vision: today it's one flat, meaning-searchable notebook with smart routing. The end goal — a router orchestrating swappable per-topic expert memories — isn't built yet; the buckets are just labels for now. This proves the search + routing + honesty floor first.
